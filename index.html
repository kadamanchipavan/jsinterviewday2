1.What is creation phase and execution phase?

Compiler runs through the entire code for 2 time before actually executing the code creation phase=>
  • In the first run, It picks all function declarations and stores them in memory with their reference
  • In the second run, It picks all variables and assign undefined to them. In the event of a conflict between variable and function declaration name then that variable is ignored.
    • execution phase=> Finally, right after the creation phase of an Execution Context comes the execution phase. This is the stage where the actual code execution begins, Variables assigned with values and Functions executed.

    7.what are call backs

    Function that is passed as an argument to another function is known as callback function. As JS is event driven language which means they don't stop the program from running until they are completed 
            function callback(greeting){
            console.log("congratulations");
            greeting();
        };
        function show(){
